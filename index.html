This project revolves around one of the most famous datasets of all time, then Enron corpus. Enron, an american energy company was at the center of one of the largest cases of fraud in history. Many executives at Enron were indicted for a variety of charges and some were later sentenced to prison.

The purpose of this project is to showcase the machine learning concepts and techniques learned throughout the Udacity course. To do this, I will attempt to build a classifier that can identify persons of interest (POIs) by training on data made public through the Enron email and financial datasets.

Data Cleaning¶

As we see above, the number of NaN values is a little shocking. I turned to reading the documentaion "enron61702insiderpay.pdf" that provides the data schema and decided to split the NaN values into two categories (Financial & Email) that I will cleanse differently.

Financial data should be "0" instead of NaN, so I will replace those in the dataset.

Email data of 60 people is missing for:

  • "from_messages"
  • "from_poi_to_this_person"
  • "from_this_person_to_poi"
  • "shared_receipt_with_poi"
  • "to_messages"

35 people don't have data for email_address, leaving 25 people with an email_address but no email data. I will assume that these 60 people are simply missing data on their email activity (rather than employees never using email) and give them the median of each email feature.

# Create function for easy conversion into Pandas DataFrame with proper format
def allFeaturesFormat(data_dict):
    
    temp_list = []
    
    for name, features_values in data_dict.items():
        temp_dict = {}
        temp_dict['name'] = name     
        for feature, value in features_values.items():
            if feature in ['poi', 'email_address']:
                temp_dict[feature] = value
            else:
                temp_dict[feature] = float(value)
        temp_list.append(temp_dict)
    df = pd.DataFrame(temp_list)
    
    return df

Engineering Features¶

"As part of the assignment, you should attempt to engineer your own feature that does not come ready-made in the dataset -- explain what feature you tried to make, and the rationale behind it. (You do not necessarily have to use it in the final analysis, only engineer and test it.)"

[relevant rubric item: “create new features”]

I decided to engineer three new features:

  • fraction_from_poi: ratio of the messages from POI to this person against all the messages sent to this person.

  • fraction_to_poi: ratio from this person to POI against all messages from this person

  • total_compensation: the sum of total_payments and total_stock_value

I landed on these features because co-conspirators in fraud tend to make a ton of money and talk to each other more often than people outside of their circle.

# Calculate fraction of messages to/from POIs
def computeFraction(poi_messages, all_messages):
    
    fraction = 0
    
    if poi_messages != 0 and all_messages != 0:
        fraction = poi_messages / all_messages
    
    return fraction

# Calculate total compensation
def computeTotal(total_payments, total_stock_value):
    
    total = total_payments + total_stock_value
    
    return total

def feature_scores(features, labels, cv, selector, score_attr):
    
    feature_names = features.columns.values
    feature_scores = defaultdict(list)
    
    for train_indices, test_indices in cv.split(features, labels):
        
        features_train = features.values[train_indices]
        features_test = features.values[test_indices]
        labels_train = labels.values[train_indices]
        labels_test = labels.values[test_indices]

        slct = selector
        slct.fit(features_train, labels_train)
        
        for feature_name, feature_score in zip(feature_names, getattr(slct, score_attr)):
            feature_scores[feature_name].append(feature_score)
    
    feature_scores = pd.DataFrame(feature_scores)
    feature_scores_avg = feature_scores.mean()
    feature_scores_avg = feature_scores_avg.sort_values(ascending=False)
    
    return feature_scores_avg

# Calculate accuracy, precision, recall, f1 score and f2 score for a given model
def metrics_scores(features_n, labels, cv, clf):
    
    true_negatives = 0
    false_negatives = 0
    true_positives = 0
    false_positives = 0

    for train_indices, test_indices in cv.split(features_n, labels):

        features_train = features_n.values[train_indices]
        features_test = features_n.values[test_indices]
        labels_train = labels.values[train_indices]
        labels_test = labels.values[test_indices]

        clf.fit(features_train, labels_train)
        labels_pred = clf.predict(features_test)

        for prediction, truth in zip(labels_pred, labels_test):
            if prediction == 0 and truth == 0:
                true_negatives += 1
            elif prediction == 0 and truth == 1:
                false_negatives += 1
            elif prediction == 1 and truth == 0:
                false_positives += 1
            elif prediction == 1 and truth == 1:
                true_positives += 1

    total_predictions = true_negatives+false_negatives+false_positives+true_positives
    accuracy = (true_positives+true_negatives)/total_predictions
    precision = true_positives/(true_positives+false_positives)
    recall = true_positives/(true_positives+false_negatives)
    f1 = 2*true_positives/(2*true_positives+false_positives+false_negatives)
    f2 = (1+2*2)*precision*recall/(4*precision+recall)

    return (accuracy, precision, recall, f1, f2)

# Calculate and plot evaluation metrics vs number of features for a given model
def feature_num_scores(features, labels, feature_names_by_score, cv, clf):
    
    num_features_accuracy_scores = []
    num_features_percision_scores = []
    num_features_recall_scores = []
    num_features_f1_scores = []
    
    for n in range(len(feature_names_by_score)):
        
        # Subset n features with the n highest feature scores
        features_n = features[feature_names_by_score[:n+1]]
        
        accuracy, precision, recall, f1, _ = metrics_scores(features_n, labels, cv, clf)
        
        num_features_accuracy_scores.append(accuracy)
        num_features_percision_scores.append(precision)
        num_features_recall_scores.append(recall)
        num_features_f1_scores.append(f1)
        
    scores_df = pd.DataFrame({
            'Number of features': range(1, len(num_features_accuracy_scores)+1),
            'Accuracy': num_features_accuracy_scores,
            'Precision': num_features_percision_scores,
            'Recall': num_features_recall_scores,
            'f1': num_features_f1_scores
        })
    
    scores_df.plot(x='Number of features', xticks=range(1, len(num_features_accuracy_scores)+1),colormap='hsv')
    
    return scores_df

Validation and Evaluation¶

"What is validation, and what’s a classic mistake you can make if you do it wrong? How did you validate your analysis? Give at least 2 evaluation metrics and your average performance for each of them. Explain an interpretation of your metrics that says something human-understandable about your algorithm’s performance."

[relevant rubric items: “discuss validation”, “validation strategy”, “usage of evaluation metrics”]

Validation involves gauging a model's performance using a dataset that is independent of the training dataset. This can be a great way to predict how well the model will generalize for unexpected data.

Because of the small number of datapoints and even fewer POIs, I decided to use StratifiedShuffleSplit to randomly split the entire dataset, and use the entire dataset both for constructing and evaluating the model. This will generate more stable evaluation metrics from an imbalanced dataset, but the drawback is that the metrics will be optimistic and may perform poorly down the road.

The metrics I used to evaluate the algorithm's were accuracy, precision, recall, and f1 score. The most important are recall, precision, and f1 (a combination of the two). Recall measures how confident the algorithm is that all employees identified as a POI are correctly labeled as a POI (low false negative rate). Precision measures how confident the algorithm is that all the employees labeled as a POI are in fact a POI (low false positive rate). f1 score is the combination of recall and precision.
